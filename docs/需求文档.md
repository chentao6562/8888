# 公司管理系统 - 需求文档

## 一、项目背景

### 痛点
- 项目整体情况不明确，数据管理能力不足
- 目标与手段的迷惑，人员对于项目目的统一认识不足
- 没有数据化的抓手

### 目标
公司业务定位：**公域流量 → 私域转化** 的销售模式

通过系统赋能降低团队的信息不畅、分析滞后的影响，提升业务的 AI 赋能水平

## 三、核心模块

### 模块1：项目管理

#### 功能描述
- 支持多个项目的创建和管理
- 每个项目设置核心目标和背景信息
- 为 AI 提供项目管理依据

#### 数据模型
```
项目 (Project)
├── 项目名称
├── 核心目标
├── 项目背景
├── 创建时间
├── 状态（进行中/已完成/暂停）
└── 关联的岗位和人员
```

#### 业务流程
```
创建项目 → 设置目标 → 分配人员 → 设定指标 → 跟踪进度 → 复盘总结
```

---

### 模块2：指标管理（SMART 指标体系）

#### 功能描述
- 根据项目特性制定岗位 SMART 指标
- 结合人员特质和擅长进行分工
- 按照紧急重要四象限为项目和岗位提供指引

#### 数据模型
```
指标 (Indicator)
├── 指标名称
├── 所属项目
├── 目标值
├── 当前值
├── 完成率
├── 负责人
├── 优先级（紧急重要四象限）
└── 截止日期
```

#### 四象限分类
| 象限 | 说明 | 处理方式 |
|------|------|----------|
| 紧急且重要 | 危机、紧迫问题 | 立即处理 |
| 重要不紧急 | 规划、预防、发展 | 计划执行 |
| 紧急不重要 | 干扰、部分会议 | 委托他人 |
| 不紧急不重要 | 琐事、时间杀手 | 尽量减少 |

---

### 模块3：前端流量看板

#### 功能描述
- 展示抖音、快手、小红书、视频号等平台的流量数据
- 展示作品数据（播放量、完播率、互动率等）
- 通过外部表格导入形成基础数据源头

#### 数据来源
| 平台 | 数据类型 |
|------|----------|
| 抖音 | 播放量、点赞、评论、分享、完播率 |
| 快手 | 播放量、点赞、评论、分享 |
| 小红书 | 阅读量、点赞、收藏、评论 |
| 视频号 | 播放量、点赞、评论、转发 |

#### 数据模型
```
流量数据 (TrafficData)
├── 平台类型
├── 账号名称
├── 作品标题
├── 发布时间
├── 播放量
├── 点赞数
├── 评论数
├── 分享/转发数
├── 完播率
└── 导入批次
```

#### 业务流程
```
导出平台数据 → 上传表格 → 系统解析 → 生成看板 → AI 分析建议
```

---

### 模块4：账号与视频内容分析

#### 功能描述
- 通过外部导入视频数据
- 自动分账号、分视频、分平台进行数据统计
- 形成数据汇总及可视化看板

#### 统计维度
```
├── 按账号统计
│   ├── 总播放量
│   ├── 平均播放量
│   ├── 作品数量
│   └── 粉丝增长
├── 按平台统计
│   ├── 各平台占比
│   └── 平台效果对比
└── 按时间统计
    ├── 日/周/月趋势
    └── 最佳发布时间
```

---

### 模块5：线索管理

#### 功能描述
- 将流量与线索的数量、质量、销售的使用率和销售能力可视化
- 对现有线索进行精细化管理
- 设定管理目标，对比目标与现状的差距

#### 线索来源
| 来源 | 说明 |
|------|------|
| 私信咨询 | 平台私信主动咨询 |
| 私信转微信 | 从私信引导到微信 |
| 微信一销 | 微信首次销售成交 |
| 正价二销 | 正价产品二次销售 |

#### 数据模型
```
线索 (Lead)
├── 线索来源（平台/渠道）
├── 客户信息
├── 跟进状态
├── 负责人（员工）
├── 转化阶段
├── 创建时间
├── 最后跟进时间
└── 成交金额
```

#### 线索漏斗
```
公域曝光 → 私信咨询 → 加微信 → 一销转化 → 二销复购
   ↓           ↓          ↓         ↓          ↓
  100%       10%        5%        2%         1%
```

---

### 模块6：AI 赋能

#### 功能描述
- 对项目分析、措施建议、人员能效考评、销售建议等环节 AI 赋能
- 前端流量的 AI 诊断分析及改进意见
- 对于环节卡点 AI 给与建议
- 单一项目的经验总结复盘对今后的指导性改进

#### AI 应用场景
| 场景 | AI 能力 |
|------|---------|
| 流量诊断 | 分析播放量低的原因，给出优化建议 |
| 内容建议 | 根据数据推荐内容方向 |
| 销售建议 | 根据线索特征推荐跟进策略 |
| 绩效分析 | 分析人员能效，给出改进方向 |
| 项目复盘 | 总结经验教训，指导未来决策 |

---

### 模块7：客户展示页面（附属功能）

#### 功能描述
- 一个对客户的前端展示页面
- 展示公司服务、案例、联系方式等

| 角色 | 权限 |

## 六、技术要求

### 技术栈
- **前端**: React + TypeScript + TailwindCSS
- **后端**: Node.js + Express + Prisma
- **数据库**: MySQL
- **AI**: 接入国内大模型 API（如通义千问、文心一言等）

### 非功能需求
- 响应式设计，支持 PC 和移动端
- 数据导入支持 Excel/CSV 格式
- 支持数据导出和报表生成
